□ Azure Virtual Machine
仮想マシンの物理的なハードウェアのメンテナンスは不要だが、仮想マシン上で動作するソフトウェアの構成、修正プログラムの適用は必要。

□ 仮想マシンスケールセット
複数の仮想マシンを一つのグループとして管理し、スケジュールもしくは負荷に応じて仮想マシンを増減させることができる。

□ 仮想マシンの課金
・仮想マシンの稼働に対する課金
Azure Portalから「割り当て解除済み」状態にすることで、課金されなくなる。
WindowsやSQL serverなどがすでにインストールされたマシンではライセンス料が含まれて課金される

・ストレージ利用量に対する課金
OSイメージが保存される仮想ディスクへの課金

・ネットワーク利用量に対する課金
インバウンド通信は無料だが、アウトバウンド通信は有料である

□ 料金プラン
・重要課金
・スポットVM
Azureの未使用リソースを利用するため安価。AWSのスポットインスタンスに相当
スポットVMにはSLAが存在しない
・Azure Hybrid Benefit
ソフトウェアアシュアランスのある既存のオンプレミスのライセンスをAzureで医療する仕組み。
仮想マシンの実行コストを大幅に削減できる
・Azure RI
最大80%コスト削減。AWSにおけるリザーブドインスタンス
・Azure開発/テスト価格
Visual Studioのサブスクリプションの所有者限定で開発テスト用の従量課金
・専用ホスト
Azureに専用の物理サーバーを用意して仮想マシンをホストする。Azureに専用のプライベートクラウドの作成ができるがその分割高

□ Azure DevTest Labs
仮想マシンとPaaSリソースの自己管理が可能。事前に構成したARMテンプレートから承認足にAsureリソースが利用できる。
今まで構成に時間構っていた作業が数分で環境を構築でき、削除も簡単に行える。テスト環境がすぐに利用できる。
